# Лекция 5. Начало второго блока: Визуал, интерфейсы и знакомство с Claude Code

## 1. Введение во второй блок
Сегодня мы начинаем второй блок обучения, который посвящен визуалу, разработке лендингов, фронтенду и созданию пользовательских интерфейсов. В рамках этого мы изучим новые, мощные инструменты для автоматизации разработки.

## 2. Знакомство с Claude Code
В отличие от Antigravity, который является полноценной и настроенной средой разработки, **Claude Code (Claude CLI/Agent)** — это инструмент от Anthropic, работающий прямо из командной строки (терминала).

**Особенности Claude Code:**
* **Скорость и мощь:** Позволяет вести разработку без необходимости вручную нажимать кнопку «Run» для каждого действия терминала.
* **Безопасность:** В отличие от некоторых открытых аналогов (например, OpenDevin), инфраструктура от Anthropic более безопасна — она не удаляет системные файлы случайно и контролируется лучше.
* **Масштабируемость:** Может стать вашим «Джарвисом»: личным ассистентом, доступным даже с телефона, который выполняет как технические, так и рутинные (маркетинг, контент) задачи.

**Сравнение с Antigravity:**
* *Antigravity:* Отлично настроен для пошаговой разработки, имеет визуальный интерфейс контроля, защищает от фатальных ошибок (можно отменить изменения). Идеально подходит как «рабочая лошадка».
* *Claude Code:* Дает большую свободу, не требует подтверждений (что ускоряет работу, но требует четкой постановки задач, иначе можно испортить код), позволяет автоматизировать рутину за рамками кодинга. Потребляет токены через API.

## 3. Мультиагентная система (Рей агентов)
В последнем масштабном обновлении Claude Code появилась поддержка мультиагентности. Теперь вы можете делегировать задачу **рою агентов**, которые будут работать не последовательно, а **параллельно**.

*Пример использования:* 
Одному агенту поручается фронтенд, второму — бэкенд, третьему — написание текстов, четвертому — SEO-оптимизация. 
Или можно запустить 5 агентов для аудита проекта:
1. Поиск потенциальных багов.
2. Оценка процента покрытия кода (Code Coverage).
3. Описание текущего функционала проекта.
4. Выявление зон для доработки.
5. Оценка проекта глазами Senior Fullstack-разработчика.

*Всё это происходит параллельно в фоновом режиме экономя колоссальное количество времени.*

## 4. MCP (Model Context Protocol)
MCP — это метод или протокол быстрого подключения ИИ к внешним сервисам. 
Например, **Context 7 (MCP)** позволяет агенту получать самую свежую документацию по API и фреймворкам до того, как он начнет писать код. Это решает извечную проблему устаревших знаний у нейросетей. Если документация обновилась, агент через MCP мгновенно это учтет. То же самое касается быстрой интеграции с GitHub, базами данных и т.д.

## 5. Skills (Навыки) v2.0 и Skill Creator
В Claude Code навыки (Skills) перешли на новый уровень:
* **Умное потребление:** Агент сам понимает, когда ему нужен скилл, и читает его описание перед полным запуском, что экономит токены. Когда выйдет более умная модель (например, Opus 5), ненужные скиллы будут автоматически отключены.
* **Глобальные и локальные скиллы:** Скиллы можно ставить глобально (доступны во всех проектах, помечаются `~`) или локально (только для конкретного проекта).
* **Skill Creator:** Встроенный инструмент, который не просто создает навык по вашему промпту, а генерирует сразу несколько вариантов, проводит **A/B тестирование** самостоятельно и выбирает самый эффективный вариант. Более того, скиллы могут самообучаться на основе ваших правок.

## 6. Управление голосом
В процессе лекции упоминался инструмент *Glider* (и другие расширения) для голосового ввода промптов прямо в интерфейс Antigravity или Claude Code, что позволяет диктовать команды, не набирая их вручную.
