## 🔍 ЭТАП 1: DISCOVERY & PROBLEM FIT
**Цель:** Найти реальную проблему, за решение которой люди готовы платить

### Что делаем:
- **Глубинные интервью (CustDev)** — минимум 20-30 разговоров с потенциальными клиентами
- **Валидация боли** — проверяем, реальна ли проблема или это "выдуманная" потребность
- **Jobs To Be Done (JTBD)** — понимаем, какую "работу" нанимает клиент
- **Анализ существующих решений** — как люди решают проблему сейчас

### Ключевые вопросы:
1. Какую проблему мы решаем?
2. Для кого эта проблема критична?
3. Сколько они платят за текущее решение?
4. Почему текущие решения не работают?

### Артефакты:
- [ ] Customer Interview Script
- [ ] Problem Statement Canvas
- [ ] JTBD Framework
- [ ] ICP (Ideal Customer Profile)

### AI-инструменты:
- AI-транскрибация интервью
- AI-анализ паттернов ответов

## 💰 ЭТАП 4: UNIT ECONOMICS
**Цель:** Просчитать финансовую модель бизнеса

### Ключевые метрики:
| Метрика | Описание | Формула |
|---------|----------|---------|
| **CAC** | Стоимость привлечения клиента | Marketing Spend / New Customers |
| **LTV** | Пожизненная ценность клиента | ARPU × Average Lifetime |
| **LTV/CAC** | Соотношение | Должно быть > 3 |
| **Payback Period** | Срок окупаемости CAC | CAC / Monthly Revenue per Customer |
| **Churn Rate** | Отток клиентов | Lost Customers / Total Customers |
| **MRR/ARR** | Регулярная выручка | Monthly/Annual Recurring Revenue |
| **Gross Margin** | Маржинальность | (Revenue - COGS) / Revenue |

### Модели монетизации:
1. **Подписка (SaaS)** — ежемесячная/годовая плата
2. **Транзакционная** — % от сделки или фикс за операцию
3. **Freemium** — бесплатный базовый + платный PRO
4. **Enterprise** — кастомные решения для бизнеса
5. **Marketplace** — комиссия с двух сторон

### Артефакты:

- [ ] Pricing Strategy
- [ ] Revenue Projections (12 месяцев)

## 🛠️ ЭТАП 5: PRODUCT & MVP
**Цель:** Создать минимально жизнеспособный продукт и протестировать его

### Принципы MVP:
1. **Минимум функций** — только core value
2. **Максимум обучения** — каждая фича должна давать данные
3. **Быстрый запуск** — недели, не месяцы
4. **Итерации** — постоянное улучшение на основе фидбека

### Типы MVP:
- **Landing Page MVP** — тестируем спрос до разработки
- **Concierge MVP** — делаем вручную то, что потом автоматизируем
- **Wizard of Oz MVP** — фронтенд есть, бэкенд — человек
- **Functional MVP** — рабочий продукт с core-фичами

### Product-Market Fit индикаторы:
- > 40% пользователей "очень расстроятся" без продукта
- Органический рост (сарафан)
- Retention > 40% после 90 дней
- NPS > 50

## 🔄 ЭТАП 6: HADI-ЦИКЛЫ
**Цель:** Систематически тестировать гипотезы и улучшать продукт

### HADI Framework:
```
┌─────────────────────────────────────────────────┐
│              HADI ЦИКЛ                          │
├─────────────────────────────────────────────────┤
│                                                 │
│   H - HYPOTHESIS (Гипотеза)                     │
│   ├── Что мы предполагаем?                      │
│   └── Какой результат ожидаем?                  │
│                                                 │
│   A - ACTION (Действие)                         │
│   ├── Что конкретно делаем?                     │
│   └── За какой срок?                            │
│                                                 │
│   D - DATA (Данные)                             │
│   ├── Какие метрики собираем?                   │
│   └── Как измеряем успех?                       │
│                                                 │
│   I - INSIGHTS (Инсайты)                        │
│   ├── Что узнали?                               │
│   └── Что меняем на основе данных?              │
│                                                 │
└─────────────────────────────────────────────────┘
```

### Бэклог гипотез:
| Приоритет | Гипотеза | ICE Score | Статус |
|-----------|----------|-----------|--------|
| P0 | Высокий impact, лёгкая реализация | 8+ | В работе |
| P1 | Средний impact | 5-7 | Очередь |
| P2 | Низкий приоритет | <5 | Backlog |

### ICE Scoring:
- **I**mpact (1-10) — влияние на ключевую метрику
- **C**onfidence (1-10) — уверенность в результате
- **E**ase (1-10) — лёгкость реализации

## 💵 ЭТАП 8: ПРОДАЖИ & ВОРОНКИ
**Цель:** Конвертировать лидов в платящих клиентов

> ⚠️ **ВАЖНО:** Маркетинг ≠ Продажи!
> - Маркетинг = генерация спроса, лидов
> - Продажи = конвертация лидов в деньги

### Структура воронки продаж:
```
┌─────────────────────────────────────────────────────────────┐
│                    ВОРОНКА ПРОДАЖ                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  LEAD (Лид)                                                 │
│  └── Человек оставил контакт / проявил интерес              │
│                                                             │
│  MQL (Marketing Qualified Lead)                             │
│  └── Лид прошёл квалификацию маркетинга                     │
│                                                             │
│  SQL (Sales Qualified Lead)                                 │
│  └── Лид готов к разговору с продажами                      │
│                                                             │
│  OPPORTUNITY (Возможность)                                  │
│  └── Идёт переговорный процесс                              │
│                                                             │
│  CUSTOMER (Клиент)                                          │
│  └── Оплатил                                                │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### Элементы системы продаж:
1. **CRM-система** — ведение базы, pipeline, аналитика
2. **Скрипты продаж** — что говорить на каждом этапе
3. **Квалификация лидов** — BANT, MEDDIC, GPCT
4. **Коммерческие предложения** — шаблоны КП
5. **Договоры и оплаты** — юридические документы

### Модели продаж:

# 💡 ПРИНЦИПЫ AI БИЗНЕС-АРХИТЕКТОРА
